River Trail

Bringing parallelism to web applications

The goal of Intel Lab’s River Trail project is to enable data-parallelism in web applications. River Trail gently extends JavaScript with simple deterministic data-parallel constructs that are translated at runtime into a low-level hardware abstraction layer. By leveraging multiple CPU cores and vector instructions, River Trail achieves significant speedup over sequential JavaScript.

For more information on River Trail, please see our wiki pages.

Using River Trail

We have an interactive River Trail shell that you can use to get accustomed to the API. If you are using Firefox, you can start right away. No installation required. However, you won't see any speedup as the interactive shell uses a sequential library implementation of the API.

Beyond playing with the API, you can play with one of our demos:

  • the particle simulation shown at IDF 2011
  • our liquid resizing demo first shown during Research @ Intel day
Our current prototype compiler for River Trail is implemented as a Firefox extension, which is required for the demos. Details on how to get it can be found in our wiki.

Getting involved

You can download the River Trail source to be used in your own demos either in zip or tar formats.
